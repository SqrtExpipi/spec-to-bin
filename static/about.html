Generate reproducible test binary files from shared JSON definitions

Spec to BIN is a free, open-source browser tool for creating fixed-layout binary data without manually calculating every offset and Hex value.

Open the web/PWA app Download the offline HTML release Read the user guide

Designed for real test-data workflows

  • Communication packets for custom TCP/UDP protocols
  • Embedded configuration and initialization data
  • EEPROM images and fixed-layout device settings
  • Repeatable test fixtures shared by development and test teams

Share the definition, reproduce the same bytes

Define fields, integer types, byte lengths, endian, encodings, padding, fixed values, and expected offsets in JSON. Store the definition in Git or another version-control system. The same JSON definition processed by the same Spec to BIN version generates the same binary output.

The GUI validates ranges, fixed-length strings, Shift_JIS and UTF-8 byte counts, IPv4 values, unresolved review markers, and expected offsets before BIN export.

Local-first and available offline

Templates, field values, and generated binaries are processed locally in the browser. The app does not upload them and has no telemetry. Install the web version as a PWA in Chrome or Edge, or download the self-contained HTML release for offline and restricted-network environments.

Optional AI-assisted specification workflow

An included prompt can help an external chat AI turn a binary specification into a draft JSON definition. Spec to BIN does not contain an AI API. A person reviews the draft, while deterministic validation and byte generation remain in the browser.

Common questions

Can I generate test BIN files without an internet connection?
Yes. Download the release ZIP and open the self-contained offline HTML file.
Can a team reproduce the same binary?
Yes. Share the JSON definition and use the same Spec to BIN version.
Are specifications or generated files uploaded?
No. Spec to BIN processes them in the browser. If you separately send a specification to an external AI, that service has its own data-handling policy.
Is this a Hex editor for reverse engineering?
No. It is focused on generating new binary data from a known structure.
Which formats are supported?
Integer types from 8 to 64 bits, raw bytes, fixed-length ASCII/UTF-8/Shift_JIS strings, IPv4, and padding are supported in v0.1.

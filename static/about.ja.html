Spec to BIN English 日本語 アプリを開く GitHub

共有JSON定義から、再現可能なテスト用バイナリファイルを生成

Spec to BINは、OffsetやHexをすべて手計算せず、固定長バイナリデータを作成するための無料オープンソース・ブラウザツールです。

Web/PWA版を開く オフラインHTML版を取得 利用ガイドを読む

実務のテストデータ作成を想定

  • 独自TCP/UDPプロトコルの通信電文
  • 組込み機器の設定・初期設定データ
  • EEPROMイメージや固定長の装置設定
  • 開発・テスト担当者が共有する再現可能なテストBIN

定義を共有し、同じbyte列を再現

フィールド、整数型、byte長、Endian、文字コード、Padding、固定値、期待OffsetをJSONで定義します。JSONをGitなどで管理すれば、同じSpec to BINバージョンと同じ定義から同じBINを生成できます。

BIN保存前に、数値範囲、固定長文字列、Shift_JIS・UTF-8のbyte数、IPv4、未確認項目、期待OffsetをGUIで検証できます。

ローカル処理とオフライン利用

JSON、入力値、生成BINはブラウザ内で処理し、アプリからアップロードしません。テレメトリもありません。ChromeまたはEdgeからPWAとしてインストールできるほか、閉域・オフライン環境向けの単一HTML版も提供しています。

外部AIで仕様書からJSONのたたき台を作成

同梱プロンプトを任意の外部チャットAIへ渡し、バイナリ仕様書からJSON定義のたたき台を作成できます。Spec to BIN自体にAI APIはありません。人間が定義を確認し、検証とbyte列生成はブラウザ内で決定的に実行します。

よくある質問

インターネットに接続せずテスト用BINを作れますか？
はい。ReleaseのZIPを展開し、自己完結型のHTMLファイルを開いて利用できます。
チームで同じBINを再現できますか？
はい。同じJSON定義を共有し、同じSpec to BINバージョンを使用してください。
仕様書や生成ファイルはアップロードされますか？
Spec to BINからはアップロードしません。別途、仕様書を外部AIへ渡す場合は、そのサービスのデータ取扱方針を確認してください。
既存BINを解析するHex Editorですか？
いいえ。既知の構造定義から新しいバイナリデータを生成する用途に特化しています。
どの型と文字コードに対応していますか？
v0.1では8～64bit整数、bytes、ASCII・UTF-8・Shift_JIS固定長文字列、IPv4、paddingに対応しています。
